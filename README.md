# Stock-Claw 智能投研系统

Stock-Claw 是一个基于 Node.js 和 React 的本地化智能投研系统。它结合了实时行情监控、量化策略筛选和 LLM（大语言模型）深度分析，旨在为个人投资者提供一套私有化、可定制的辅助决策工具。

## 🚀 核心功能 (What We Can Do)

### 1. 智能投研 Agent (StockClaw Agent)
这是系统的核心大脑，提供基于自然语言的交互式投研服务：
- **动态股票池**: 每日自动从全市场筛选高换手率、高量比的活跃个股（Top 100），构建动态观察池。
- **量化策略筛选**: 支持通过自然语言指令（如“查找连续上涨3天的股票”、“寻找MACD金叉的股票”）进行实时筛选。
- **深度技术分析**: 集成 MA (5/10日均线) 和 MACD 指标，自动生成包含 K 线数据、技术形态诊断和交易建议的深度分析报告。
- **本地化数据**: 采用 SQLite 本地数据库存储 K 线数据，支持毫秒级回测和筛选，无需频繁请求外部 API，避免封控和延迟。

### 2. 实时行情看板
- **自选股监控**: 实时刷新自选股的价格、涨跌幅和成交量。
- **资金流向**: 监控北向资金、行业板块资金流向，捕捉市场热点。
- **市场快讯**: 聚合选股宝等平台的实时快讯，并通过 AI 提取关联个股。

## 🔌 API 接口调用 (API Reference)

后端服务运行在 `http://localhost:3001`，提供了一套 RESTful API 供前端或第三方工具调用。

### 基础信息
- **Base URL**: `http://localhost:3001`
- **Response Format**: JSON

### 核心接口

#### 1. 获取动态股票池
获取当前系统根据量化策略（如活跃度）筛选出的动态股票池列表。
- **Endpoint**: `GET /api/stocks/pool`
- **Response**:
  ```json
  {
    "code": 0,
    "data": [
      { "code": "sz002261", "name": "拓维信息" },
      ...
    ]
  }
  ```

#### 2. AI 投研对话
与 StockClaw Agent 进行对话，执行选股或分析任务。
- **Endpoint**: `POST /api/agent/chat`
- **Body**:
  ```json
  {
    "message": "帮我找最近连涨3天的股票"
  }
  ```
- **Response**: 返回包含分析结果的 Markdown 文本流。

#### 3. 手动触发数据同步
强制系统从外部数据源（如东方财富）同步最新的 K 线数据到本地数据库。
- **Endpoint**: `POST /api/sync/trigger`
- **Response**: `{"status": "ok", "message": "Sync started"}`

#### 4. Agent Manifest
提供符合 OpenClaw 标准的工具描述文件，可直接被其他 AI Agent 平台集成。
- **Endpoint**: `GET /manifest.json`

## 🤖 如何使用 AI 服务 (AI Guide)

StockClaw 内置了强大的 AI 服务，您可以通过以下方式使用：

### 1. 前端交互 (推荐)
1. 启动项目后，访问前端页面 `http://localhost:5173`。
2. 点击侧边栏的 **"Agent Chat"** 进入对话界面。
3. 在输入框中输入您的需求，例如：
   - **选股指令**: “帮我找最近连涨3天的票”、“筛选 MACD 金叉且红柱放大的股票”。
   - **分析指令**: “分析一下拓维信息 (sz002261) 的走势”。
   - **综合指令**: “找出今天涨停的股票，并分析它们的均线形态”。

### 2. 幕后机制
当您发送指令时，系统会按以下流程工作：
1. **意图识别**: LLM 分析您的自然语言，提取筛选条件（如策略类型、参数）。
2. **本地量化计算**: 后端 `ScreenerService` 在本地 SQLite 数据库中遍历股票数据，计算 MA、MACD 等指标。
3. **数据生成**: 筛选出符合条件的股票，并提取其最近 10 天的 K 线数据。
4. **深度分析**: LLM 基于筛选结果和 K 线数据，生成包含数据表格、技术诊断和操作建议的 Markdown 报告。

### 3. 数据更新
系统设计为**每日自动更新**。但在盘中或数据滞后时，您可以：
- 访问 **Dashboard** 页面。
- 点击右上角的 **"刷新数据"** 按钮（或通过 API 触发），系统将自动拉取最新的 K 线数据存入本地数据库，确保分析结果的时效性。
